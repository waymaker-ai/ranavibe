# Compliance Framework Reference

## HIPAA (Health Insurance Portability and Accountability Act)
### 18 Protected Health Information (PHI) Identifiers:
1. Names
2. Geographic data smaller than a state
3. Dates (except year) related to an individual
4. Phone numbers
5. Fax numbers
6. Email addresses
7. Social Security Numbers
8. Medical record numbers
9. Health plan beneficiary numbers
10. Account numbers
11. Certificate/license numbers
12. Vehicle identifiers and serial numbers
13. Device identifiers and serial numbers
14. Web URLs
15. IP addresses
16. Biometric identifiers
17. Full-face photographs
18. Any other unique identifying number

### Required Disclaimer:
"This is not medical advice. Consult a healthcare professional for medical guidance."

## GDPR (General Data Protection Regulation)
### Key Principles:
- Data minimization: only collect/process what's necessary
- Purpose limitation: only use data for stated purposes
- Consent: inform users and obtain consent for data processing
- Right to erasure: users can request deletion of their data
- Data portability: users can request their data in a portable format
- Breach notification: report breaches within 72 hours

## SEC (Securities and Exchange Commission)
### Key Rules:
- Never provide specific investment advice without proper licensing
- Never guarantee investment returns
- Always include forward-looking statement disclaimers
- Never use material nonpublic information
- Required: "Past performance does not guarantee future results."

## PCI DSS (Payment Card Industry Data Security Standard)
### Key Rules:
- Never display full credit card numbers
- Never store CVV/CVC codes
- Mask card numbers: show only last 4 digits
- Never log sensitive authentication data

## FERPA (Family Educational Rights and Privacy Act)
### Key Rules:
- Student records require parental/student consent before disclosure
- Directory information exceptions are limited
- Financial aid information is protected
- Disciplinary records are protected

# RANA 3.0 — AI Governance Rules Engine

**Date:** 2026-02-26
**Status:** Approved
**Authors:** Ashley Kays, Claude

---

## Vision

RANA 3.0 transforms from a general-purpose AI development framework into a focused **AI governance rules engine**. One command scans your repository and generates security, compliance, and project rules for every AI coding tool — Claude Code, Cursor, GitHub Copilot, Continue.dev, Gemini CLI, and any MCP client.

**The pitch:**

> RANA — AI Governance Rules Engine
>
> Generate security, compliance, and project rules for every AI coding tool. One command, every tool.
>
> ```bash
> npx @rana/cli init
> ```

---

## Problem Statement

1. **48-62% of AI-generated code contains security vulnerabilities** (2025-2026 data)
2. **No tool generates governance rules automatically** — CLAUDE.md, .cursorrules, copilot-instructions are all manual
3. **70% of enterprises want compliance-as-code** but no developer-actionable tooling exists
4. **Developers' #1 request is "improved contextual understanding"** from AI tools (26% in surveys)
5. **AI configuration is fragmented** — each tool has its own format, none include security/compliance presets

## Rules Engine Design

### The Pipeline

```
Repository -> Scanner -> Rules (.rana/rules.yml) -> Compiler -> Tool Configs
```

### Scanner

Automatically detects:
- **Frameworks**: Next.js, React, Express, Fastify, etc.
- **Structure**: Monorepo vs single, directory layout, key files
- **Conventions**: Import patterns, naming conventions, testing setup
- **Dependencies**: Package manager, key libraries, versions
- **CI/CD**: GitHub Actions, Vercel, etc.
- **Existing configs**: tsconfig, eslint, prettier, etc.

### Rules Format (`.rana/rules.yml`)

```yaml
version: "3.0"
name: "my-project"

context:
  description: "Healthcare patient portal"
  architecture: monorepo
  frameworks: [next.js, prisma, tailwind]
  structure:
    src/app: "Next.js app router pages"
    src/lib: "Shared utilities and helpers"
    src/components: "React components"
    prisma/schema.prisma: "Database schema"
  conventions:
    - "Use server components by default"
    - "All API routes require authentication"
    - "Use Zod for all input validation"

security:
  level: enterprise  # minimal | standard | enterprise | paranoid
  rules:
    - id: no-sql-injection
      severity: critical
      pattern: "Never use string concatenation in SQL queries"
      detect: "string interpolation in database query functions"
      fix: "Use parameterized queries or ORM methods"
    - id: no-secrets-in-code
      severity: critical
      pattern: "Never hardcode API keys, passwords, or secrets"
      detect: "literal strings matching API key patterns"
      fix: "Use environment variables"
    - id: input-validation
      severity: high
      pattern: "All user input must be validated with Zod schemas"
    - id: no-unsafe-code-execution
      severity: high
      pattern: "Never use dynamic code execution patterns"

compliance:
  presets: [hipaa, gdpr]
  custom:
    - id: pii-handling
      standard: HIPAA
      pattern: "All PII must be encrypted at rest and in transit"
    - id: audit-logging
      standard: HIPAA
      pattern: "All data access must be logged with user, timestamp, and action"
    - id: data-retention
      standard: GDPR
      pattern: "Include deletion capability for all user data"

workflow:
  testing: "pnpm test"
  linting: "pnpm lint"
  building: "pnpm build"
  deploying: "Deploy via Vercel CLI or GitHub Actions"
  branching: "feature/* -> main via PR"
  pr-rules:
    - "All PRs require security scan passing"
    - "All PRs require at least one approval"

governance:
  models:
    preferred: [claude-sonnet-4-20250514, gpt-4o]
    forbidden: [claude-sonnet-4-5-20250929]
    reasoning: "Use Sonnet for most tasks, Opus for complex architecture"
  cost:
    budget: 100  # USD per month
    alerts: [50, 80, 95]
  quality:
    min-test-coverage: 80
    require-types: true
    max-complexity: 15
```

### Compiler Targets

The rules engine compiles `.rana/rules.yml` into:

| Target | Output File | For Tool |
|--------|-------------|----------|
| Claude MD | `CLAUDE.md` | Claude Code |
| Cursor Rules | `.cursorrules` | Cursor |
| Copilot Instructions | `.github/copilot-instructions.md` | GitHub Copilot |
| Continue Rules | `.continue/rules/rana.md` | Continue.dev |
| Agents MD | `AGENTS.md` | Universal AI agents |
| Gemini MD | `GEMINI.md` | Gemini CLI |

Each compiler translates the structured YAML into the natural-language format each tool expects, including all security/compliance rules as actionable instructions.

## CLI Commands

```bash
# Initialize — scan repo and generate rules
rana init [--preset hipaa,gdpr] [--tool claude,cursor,copilot]

# Scan — check codebase against rules
rana scan [path] [--fix] [--json] [--fail-on critical]

# Sync — regenerate tool configs from rules.yml
rana sync [--tool claude,cursor,copilot,all]

# Score — security/compliance score report
rana score [--json] [--html] [--verbose]

# Presets — manage compliance/framework presets
rana preset add <name>    # hipaa, gdpr, soc2, nextjs, react...
rana preset list
rana preset remove <name>

# Diff — show changes since last sync
rana diff

# Doctor — check installation health
rana doctor

# Upgrade — update rules for new RANA versions
rana upgrade
```
